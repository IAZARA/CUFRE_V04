Desarrollar un sistema de gestión de expedientes que permita administrar, dentro de cada expediente, entidades asociadas como personas, domicilios, medios de comunicación, fotografías y documentos, centralizando toda la información relevante de cada caso.
El sistema debe estar enfocado exclusivamente en la gestión de expedientes, permitiendo la carga, consulta y obtención de estadísticas sobre los mismos, utilizando Oracle como base de datos, Spring Boot para el backend y el frontend más adecuado para una experiencia de usuario ágil, intuitiva y eficiente.
Todos los nombres de tablas y campos en la base de datos deben estar en mayúsculas, siguiendo las mejores prácticas de integración con Oracle.

3. Relaciones y Flujo de Carga
El usuario crea un EXPEDIENTE y carga todos los datos posibles.
En la pestaña PERSONAS, vincula personas (profugo, familiares, asociados) usando una tabla intermedia si es necesario (PERSONA_EXPEDIENTE).
En la pestaña DOMICILIOS, agrega domicilios del profugo (o de otras personas).
En la pestaña MEDIOS DE COMUNICACION, agrega medios para cada persona.
En la pestaña DOCUMENTOS y FOTOGRAFIAS, sube archivos asociados al expediente.
Todos los campos de tipo enum se presentan como selectores en la UI.
Todos los campos booleanos se presentan como checkboxes.
El sistema permite consultar, editar y obtener estadísticas de los expedientes.
